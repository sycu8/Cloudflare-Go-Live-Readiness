GitHub Action

Run cf-ready scan in CI on every pull request. Upload SARIF security findings and fail when blockers are present.

Quick start

name: CF Ready

on:
  pull_request:
    branches: [main]

jobs:
  readiness:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      security-events: write
    steps:
      - uses: actions/checkout@v4
      - uses: sycu8/Cloudflare-Go-Live-Readiness/action@main
        with:
          cwd: .
          fail-on-blocker: "true"

Inputs

  • cwd — project directory to scan (default .)
  • config — optional path to cf-ready.config.json
  • fail-on-blocker — fail when blockers exist (default true)
  • cf-ready-root — path to cf-ready package when vendoring (default repo root)

Outputs

  • exit-code — 0 ready, 1 blockers, 2 error
  • production-ready, blocker-count, overall-score

The action uploads security-findings.sarif and a JSON scan artifact when the scan completes.
