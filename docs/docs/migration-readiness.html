Migration readiness for Cloudflare deployments

CF Ready inspects your repository for Cloudflare runtime blockers and framework-specific migration paths.

What we detect

  • Node.js APIs incompatible with Workers (fs, child_process, native modules)
  • Next.js router patterns, middleware, and vinext / OpenNext recommendations
  • Vite and React SPA → Cloudflare Pages or Workers Assets
  • Express and Node servers → progressive migration via Hono
  • Current hosting target (Vercel, Netlify, etc.)

Commands

cf-ready scan
cf-ready migration-plan
cf-ready inspect --json

Migration findings are weighted at 35% of the overall score. See Scoring.
