Security readiness before go-live

Security checks run read-only against your source tree and dependencies.

Checks include

  • Exposed .env files and missing .gitignore rules
  • Hardcoded secrets, API keys, JWT secrets, database URLs
  • Unsafe CORS patterns and risky public files
  • Source maps in public output
  • Dependency audit when npm audit is available

Commands

cf-ready security-scan
cf-ready scan

Outputs SARIF at security-findings.sarif for CI and GitHub code scanning.
